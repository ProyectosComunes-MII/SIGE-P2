Introducción al problema¶

En esta segunda pr√°ctica vamos a resolver un problema de aprendizaje autom√°tico consistente en predecir qu√© prenda de ropa corresponde cada imagen. Para ello vamos a trabajar con el conjunto de im√°genes FASHION-MNIST que podemos importar directamente desde Keras.

En este documento se aplicar√°n t√©cnicas diferentes para construir diversos modelos con los que mejorar la calidad y el n√∫mero de aciertos. Para ello comenzaremos probando desde los m√°s sencillos hasta los m√°s complejos.

Modelos de predicción¶

En esta secci√≥n procedemos a entrenar diversos modelos de predicci√≥n con diferentes t√©cnicas. Sin embargo, en primer lugar definimos tres funciones comunes a todos ellos para entrenarlos, validarlos y graficar su evoluci√≥n durante el proceso para analizar qu√© ha ocurrido.

La primera funci√≥n denominada train_model es la encargada de llevar a cabo el entrenamiento, midiendo el tiempo invertido en este, y devolviendo el modelo conseguido m√°s el valor de esta medida. Para seguir la evoluci√≥n en directo del proceso podemos especificar un porcentaje para dividir el conjunto de im√°genes en entrenamiento y validaci√≥n mediante el par√°metro validation_split y el argumento verbose. Este √∫ltimo lo hemos eliminado de la memoria para que los logs del entrenamiento no ocupen demasiado espacio, pero lo hemos utilizado durante la realizaci√≥n de esta pr√°ctica. Asimismo, hemos establecido el par√°metro shuffle=False para que no mezcle aletoriamente el conjunto de datos y de ese modo no obtener resultados diferentes en cada ejecuci√≥n de entrenamiento [1].

A continuaci√≥n encontramos la funci√≥n plot_train con la que representar el proceso anterior de manera gr√°fica para visualizar c√≥mo ha evolucionado la tasa de error tanto en el conjunto de entrenamiento como en el de validaci√≥n tras cada √©poca.

Por √∫ltimo, validamos el modelo conseguido con la funci√≥n eval_model con el que obtenemos la tasa de acierto, de fallo, la matriz de confusi√≥n y el coeficiente de Kappa. De este modo podremos evaluar los modelos predictivos con diferentes medidas para conocer su calidad y su capacidad de clasificaci√≥n.

def train_model(model, train_imgs, train_lbs, val, batch_s, n_classes, epochs, callbs=None):
  """"Funci√≥n que entrena un modelo y mide el tiempo invertido. Para este proceso se
    debe especificar el conjunto de entrenamiento y las etiquetas, el tama√±o del lote,
    el n√∫mero de clases y de iteraciones, y el porcentaje reservado del conjunto de datos
    para realizar un proceso de validaci√≥n."""
  start_time = time.time()
  history = model.fit(train_imgs, train_lbs, epochs=epochs, batch_size=batch_s,
              verbose=0, validation_split=0.3, shuffle=False, callbacks=callbs)
  elapsed_time = time.time() - start_time
  
  print("\nTiempo de entrenamiento:", round(elapsed_time, 2), "segundos.")
  return history

def eval_model(model, test_images, test_labels, eval_cnn):
  """Funci√≥n para evaluar los modelos obteniendo la tasa de aciertos, fallos, 
    la matriz de confusi√≥n y el coeficiente de Kappa"""
  score = model.evaluate(test_images, test_labels, verbose=0)
  test_predicted = model.predict(test_images)
  # Si evaluamos modelos CNN no hace falta transformar las etiquetas de test.
  if (not eval_cnn):
    test_labels = test_labels.argmax(axis=-1)

  test_predicted = test_predicted.argmax(axis=-1)
  # Coeficiente de Kappa
  kappa = cohen_kappa_score(test_labels, test_predicted)
  print('-----------------Test-----------------')
  print('Tasa de aciertos en test:', round(score[1]*100, 2), '%')
  print('Tasa de errores en test:', round((100 - score[1]*100),2), '%')
  print('Coeficiente de Kappa:', round(kappa*100, 2), '%')
  # Dibujamos la matriz de confusi√≥n
  conf_matrix = confusion_matrix(test_labels, test_predicted)
  plt.imshow(conf_matrix, cmap=plt.cm.Blues)
  plt.xlabel("Predicciones")
  plt.ylabel("Reales")
  plt.xticks([], [])
  plt.yticks([], [])
  plt.title('Matriz de confusi√≥n')
  plt.colorbar()

Modelo 1. Arquitectura básica¶

En este primer modelo proponemos una topolog√≠a de red convolutiva b√°sica. Para ello vamos a a√±adir una capa de entrada que acepte una imagen de 28x28 p√≠xeles, una capa oculta de 256 neuronas con una funci√≥n de activaci√≥n ReLU. Por √∫ltimo una capa de salida con tantas neuronas como etiquetas (diez) y una capa de activaci√≥n softmax, para conocer el grado de pertenencia de una imagen a los diez tipos de prendas de ropa diferentes.

Asimismo, hemos realizado diversas pruebas con distintos optimizadores pero el que mejores resultados ha proporcionado ha sido el RMSprop, por lo que es el que establecemos de momento. Este modelo se ha entrenado durante 30 √©pocas, puesto que tras diversos experimentos, podemos determinar que un mayor n√∫mero de iteraciones no ayudan a mejorar los resultados, por lo que este valor ha sido el que mejor relaci√≥n calidad~tiempo nos ha proporcionado.

Finalmente, hemos estudiado las distintas t√©cnicas de regularizaci√≥n para intentar combatir el posible sobreaprendizaje durante el entrenamiento. Keras dispone de tres m√©todos para aplicar la regularizaci√≥n L1, L2 o una combinaci√≥n de ambas. Para esta arquitectura, en particular, la que mejores resultados ha proporcionado ha sido la regularizaci√≥n L2 con una diferencia de casi un 9% comparada con L1 y la combinaci√≥n de ambas.

De igual modo hemos realizado diversos experimentos con el coste a√±adido para establecer un valor con el que ni se penalice en exceso ni que tampoco no proporcione ning√∫nefecto. A valores m√°s altos, mayor penalizaci√≥n, y eso se traduce en que nuestro modelo pierde mayor capacidad de generalizaci√≥n puesto que el coste adicional es muy alto y no ayuda a ajustar bien los pesos de la red. Por ende, vamos a apostar por un valor m√°s peque√±o como 0.00001 para conseguir un modelo m√°s ajustado.

Modelo 2. Dropout¶

En este segundo modelo vamos a introducir un mayor n√∫mero de capas ocultas adem√°s de una t√©cnica denominada dropout. Consiste en desactivar aleatoriamente un determinado porcentaje de neuronas de una capa con el objetivo de reducir el sobreajuste. Este fen√≥meno ocurre a menudo cuando existe una gran cantidad de neuronas en una capa que pueden verse influenciadas entre s√≠ y, por lo tanto, aportan informaci√≥n similar con lo que el modelo no mejora. Por ello, esta t√©cnica pretende reducir el n√∫mero de neuronas que contribuyen al entrenamiento para que realicen un mayor esfuerzo por conseguir caracter√≠sticas que mejoren la clasificaci√≥n.

Para ajustar el porcentaje de neuronas a desactivar, existen varios art√≠culos [2] que comentan las tres opciones posibles. En primer lugar podemos establecer una capa dropout en la de entrada, tambi√©n conocida como capa visible. Tras diversos experimentos, podemos determinar que tanto para valores altos como bajos, los resultados empeoran, tanto por la tasa de aciertos que se sit√∫a sobre un 83% como el coeficiente de Kappa que disminuye hasta el 81%. Por lo tanto, en nuestro modelo no vamos a introducir dropout en la capa de entrada.

Para las capas ocultas la tasa m√°s adecuada suele estar en el rango [0.2, 0.5]. Seg√∫n las diversas pruebas que hemos realizado, estableciendo el valor m√°ximo en una capa es m√°s ventajoso que en todas las capas, siendo la diferencia de 3 puntos en la tasa de acierto y el coeficiente de Kappa. Esto nos indica que restringir demasiado el uso de las neuronas de las capas ocultas provoca cierta p√©rdida de informaci√≥n que se traduce en una menor capacidad de generalizaci√≥n. Sin embargo, apostando por el valor m√≠nimo de 0.2 en todas las capas ocultas, hemos conseguido mejorar la tasa de acierto hasta alcanza aproximadamente un 89% con un coeficiente de Kappa alrededor del 88%.

Por otro lado, a√±adir m√°s capas ocultas no siempre significa obtener un mejor modelo, ya que, de nuevo, puede producirse sobreajuste y perder capacidad de predicci√≥n. Para nuestro problema, tras realizar diversos experimentos, hemos concluido que el modelo no mejora significativamente con un mayor n√∫mero de capas ocultas, sino que su comportamiento es muy similar al anterior de topolog√≠a b√°sica. De entre todos los experimentos, presentamos un segundo modelo compuesto por dos capas ocultas adicionales de 128 y 64 neuronas, respectivamente, m√°s capas dropout solo para las tres ocultas con una tasa de 0.2, ya que es la mejores resultados nos ha proporcionado hasta el momento.

Finalmente, como en el caso anterior, un mayor n√∫mero de iteraciones no se traduce en mejores resultados, por lo que continuamos estableciendo como n√∫mero de √©pocas 30.

Modelo 3. Red convolutiva¶

En este tercer modelo vamos a entrenar una red convolutiva para comprobar si, con una topolog√≠a orientada a im√°genes, conseguimos mejorar los resultados que hemos obtenido hasta el momento.

Al tratarse de una CNN, el preprocesamiento del conjunto de datos es ligeramente diferente al anterior. Mantenemos el escalado de los valores de las im√°genes, pero debemos transformarlas en vectores 3D con la siguiente estructura: (alto, ancho, n√∫mero de canales). Por ello, definimos un nuevo procesamiento que incluya esta √∫ltima modificaci√≥n para trabajar con un conjunto de datos compatible con este modelo de red.

Asimismo, tambi√©n hemos realizado varios experimentos para conocer los distintos resultados que proporcionan los optimizadores de Keras. En este caso, Adam ha sido el que mejor calidad~tiempo obtiene, por lo que para los modelos convolotuvos ser√° este el que utilicemos en lugar del RMSprop. La raz√≥n de ello reside en que este algoritmo es uno de los m√°s eficientes computacionalmente y se encuentra orientado para aplicarlo en problemas con muchos datos por su robustez en los cambios del Gradiente Descendente que utiliza [3].

Como este proceso es com√∫n al uso de los posteriores modelos convolutivos, tambi√©n hemos incluido la posibilidad de incluir la t√©cnica de aumentaci√≥n de datos, con la que generar im√°genes modificadas con respecto al conjunto original. El objetivo consiste en evitar un sobreajuste a los datos de entrenamiento proporcion√°ndole al modelo muestras de diferentes cualidades a las originales. Para ello realizaremos algunas transformaciones para cambiarlas ligeramente, de modo que al modelo le cueste m√°s identificarlas. Esto se ha realizado utilizando la biblioteca Augmentor, la cual tras definir una serie de operaciones en un Pipeline genera un determinado n√∫mero de im√°genes teniendo como base las del conjunto de entrenamiento. Estas nuevas im√°genes sufrir√°n dos transformaciones que ser√°n una rotaci√≥n hacia un lado u otro y una transformaci√≥n de tipo zoom. A continuaci√≥n se puede ver la implementaci√≥n de dicha funci√≥n.

import Augmentor as aug
from sklearn.utils import shuffle
import numpy as np
def elastic_augmentor(train_images, train_labels, number_images):
    # Creamos el Pipeline de Augmentor
    pipe = aug.Pipeline()
    # Introducimos la operaci√≥n de distorsi√≥n aleatoria, esto nos proporciona una distorsi√≥n el√°stica leve
    pipe.random_distortion(probability = 0.5, grid_width = 5, grid_height = 5, magnitude = 1)
    # Introducimos la operaci√≥n de rotaci√≥n hacia ambos lados
    pipe.rotate(probability=0.7, max_left_rotation=10, max_right_rotation=10)
    # Introducimos la operaci√≥n de zoom con un zoom entre 1.1 y 1.3
    pipe.zoom(probability=0.3, min_factor=1.1, max_factor=1.3)
    # Definimos el generador de im√°genes
    g = pipe.keras_generator_from_array(train_images, train_labels, batch_size=number_images)
    
    # Generamos tantas im√°genes y etiquetas como hayamos especificado anteriormente
    X, y = next(g)
    # Le damos al array de im√°genes obtenidas la misma forma que nuestro dataset original
    X = X.reshape((number_images, 28, 28))
    
    # Concatenamos im√°genes y etiquetas generadas con los conjuntos originales y los desordenamos
    train_images = np.concatenate((train_images,X), axis=0)
    train_labels = np.concatenate((train_labels,y), axis=0)
    train_images, train_labels = shuffle(train_images, train_labels, random_state=1)
    return [train_images, train_labels]

def load_cnn_data(augmentation=False):
  """Funci√≥n para preprocesar los dos conjuntos de datos para que sean aceptados
    por una CNN. Existe la posibilidad de aplicar la t√©cnica augmentation para 
    fabricar m√°s im√°genes con ciertas transformaciones."""
  # Obtenemos los datos originales de nuevo
  (train_images_cnn, train_labels_cnn), (test_images_cnn, test_labels_cnn) = fashion_mnist.load_data()
  # Aplicamos la t√©cnica augmentation si el argumento est√° a True
  if (augmentation):
    [train_images_cnn, train_labels_cnn] = elastic_augmentor(train_images_cnn, train_labels_cnn, 20000)
  # Escalamos los valores para normalizarlos en el rango [0,1] (train_images, train_labels), (test_images, test_labels)
  train_images_cnn = train_images_cnn.astype('float32') / 255.0
  test_images_cnn = test_images_cnn.astype('float32') / 255.0
  # Convertimos las im√°genes en vectores 3D (alto, ancho, canales)
  dim = train_images_cnn.shape[1]
  num_classes = 10
  channels = 1
  train_images_cnn = np.reshape(train_images_cnn, (train_images_cnn.shape[0], dim, dim, channels))
  test_images_cnn = np.reshape(test_images_cnn, (test_images_cnn.shape[0], dim, dim, channels))

  print("Conjunto de datos de entrenamiento para la CNN:", train_images_cnn.shape)
  print("Conjunto de datos de test para la CNN:", test_images_cnn.shape)

Modelo 6. Transfer learning¶

Como hemos podido apreciar, las mejoras conseguidas hasta el momento no han sido demasiado significantes como podr√≠amos esperar. Esto puede ser debido a que la red no aprenda las caracter√≠sticas de las im√°genes lo suficientemente bien por lo que continua cometiendo errores.

Una posibles soluci√≥n a este problema se conoce como Transfer Learning. Esta t√©cnica es de gran relevancia en el Deep learning y es que en lugar de entrenar una red neuronal desde cero, lo cual puede necesitar una gran cantidad de datos y tiempo, utilizamos un modelo ya entrenado. De este modo, podemos obtener una red pre-entrenada como punto de partida, para posteriormente, continuar entren√°ndola con nuestro conjunto de datos.

El principal objetivo reside en utilizar el conocimiento aprendido por la red neuronal para reconocer m√°s f√°cilmente los patrones de nuestros datos y ayudar as√≠ en la clasificaci√≥n de las im√°genes.

Para este proposito podemos utilizar una gran variedad de redes, y es que en Keras existen diversos modelos pre-entrenados para Visi√≥n por Computador [5]. De todos los existentes, vamos a aplicar el VGG16 inspir√°ndonos en este tutorial [6] para conocer las condiciones y el preprocesamiento que se le debe realizar al conjunto para utilizar este modelo.

En primer lugar, vamos a definir una funci√≥n comun tanto a este modelo como al siguiente en la que se transformen transformar las im√°genes del conjunto de en vectores 3D y escalarlas hasta los 48x48 p√≠xeles puesto que es el formato de entrada que acepta la red VGG16. A continuaci√≥n, normalizamos los valores como en los modelos anteriores, y los preprocesamos para codificarlos como un conjunto de im√°genes [7]. Por √∫ltimo, convertimos las etiquetas en multidimensionales para que cada fila est√© representada por diez elementos en los que todos son 0 excepto un 1 en la casilla a la etiqueta que corresponde la imagen. Este mismo procedimiento se puede realizar sobre el conjunto aumentado de im√°genes aplicando, para ello, la funci√≥n explicada anteriormente con la que realizar data augmentation.

def load_data_vgg16(augmentation):
  # Obtenemos los conjuntos originales y unificamos los p√≠xeles en un solo vector 
  # de 784 componentes
  (train_X, train_Y), (test_X, test_Y) = fashion_mnist.load_data()
  if (augmentation > 0):
    (train_X, train_Y) = elastic_augmentor(train_X, train_Y, 50000)

  train_X = train_X.reshape((train_X.shape[0], 28*28))
  test_X = test_X.reshape((test_X.shape[0], 28*28))  
  # Transformamos los vectores a 3D
  train_X = np.dstack([train_X] * 3)
  test_X = np.dstack([test_X] * 3)
  # Volvemos desplegar los p√≠xeles en 3D
  train_X = train_X.reshape(-1, 28, 28, 3)
  test_X = test_X.reshape (-1, 28, 28, 3)
  # Reescalamos las im√°genes a 48x48 para que las acepte VGG16
  train_X = np.asarray([img_to_array(
      array_to_img(im, scale=False).resize((48,48))) for im in train_X])
  test_X = np.asarray([img_to_array
      (array_to_img(im, scale=False).resize((48,48))) for im in test_X])
  print("Conjunto de entrenamiento preprocesado:", train_X.shape)
  print("Conjunto de test preprocesado:", test_X.shape)

  # Normalizamos los datos a un rango [0,1]
  train_X = train_X / 255.
  test_X = test_X / 255.
  train_X = train_X.astype('float32')
  test_X = test_X.astype('float32')
  # Preprocesamos los dos conjuntos 
  train_X = preprocess_input(train_X)
  test_X = preprocess_input(test_X)

  # Transformamos las etiquetas a binarias (one-hot encoding)
  train_Y_one_hot = to_categorical(train_Y)
  test_Y_one_hot = to_categorical(test_Y)
  print("Etiquetas de entrenamiento preprocesadas:", train_Y_one_hot.shape)
  print("Etiquetas de test preprocesadas:", test_Y_one_hot.shape, "\n")

  return [train_X, train_Y_one_hot, test_X, test_Y_one_hot]

Modelo 7. Fine Tuning¶

Por √∫ltimo vamos a utilizar otra t√©cnica denominada fine tuning que consiste en hacer uso de una red neuronal pre-entrenada con el fin de poder adaptarla a nuestro problema. Para ello se suelen a√±adir algunas capas al modelo para entrenarlas con nuestro conjunto de im√°genes, dejando las capas originales fuera de este segundo entrenamiento. As√≠, aprovechamos el conocimiento adquirido por esta red pre-entrenada para a√±adirle el asociado a nuestros datos con el que poder utilizarla para resolver este problema de clasificaci√≥n.

En nuestro caso, usaremos de nuevo el modelo VGG16 anterior al que le a√±adiremos una capa GlobalAveragePooling, que calcula el valor medio de cada caracter√≠stica identificada de la capa anterior para reducir los datos disponibles y preparar el modelo para obtener los resultados de clasificaci√≥n [10]. Como segunda capa a√±adimos una capa densa de 512 neuronas con una funci√≥n de activaci√≥n ReLU combinada con dropout. Por √∫ltimo, a√±adimos otra capa densa softmax con 10 neuronas para obtener las probabilidades de que una imagen se corresponda con las diferentes prendas disponibles. Tras experimentar con varias topolog√≠as, nos hemos inspirado en este tutorial [11] para presentar la arquitectura anterior, que es la que mejores resultados nos ha proporcionado.

En este caso, tampoco es ventajoso aplicar data augmentation principalmente por el coste computacional, temporal y de memoria. Uno de los inconvenientes de utilizar entornos cloud con recursos compartidos y limitados, es que tienes que adaptarte a ellos, salvo que desees invertir tu capital en conseguir m√°s recursos. Tras realizar varios experimentos fallidos tanto por falta de memoria RAM como por los propios resultados, podemos determinar que no merece la pena aumentar el conjunto de datos, por lo que aplicaremos fine tuning al conjunto original.

Obviamente, algunas de estas soluciones ser√°n mejores que otras, y es que en la mayor parte de las ocasiones, no hay una soluci√≥n clara a un problema. Por lo tanto, deberemos emplear una aproximaci√≥n parecida a la que hemos realizado en este trabajo para encontrar la soluci√≥n √≥ptima.

Nosotros hemos conseguido algunos resultados bastante interesantes y es que desde el modelo inicial hemos mejorado bastante los resultados obtenidos, los cuales los hemos recopilado en la siguiente tabla.

  • En primer lugar, destacamos que con un modelo muy b√°sico podemos obtener resultados bastante aceptables, tanto por la tasa de aciertos como por la calidad que demuestra el coeficiente de Kappa. Esto nos demuestra la potencia de las redes neuronales y su idoneidad para aplicarlas sobre problemas de clasificaci√≥n. Asimismo, una de sus principales caracter√≠sticas es la cantidad de par√°metros configurables de los que disponen, que te permiten ajustar el modelo a las necesidades de tu problema, como ha sido el caso de la regularizaci√≥n, con la que hemos obtenido un entrenamiento m√°s estable y de mayor calidad.

  • Un segundo factor que hemos aprendido con respecto a la topolog√≠a de la red es que, no por un mayor n√∫mero de capas y/o neuronas, vamos a obtener un modelo con mayor capacidad de generalizaci√≥n, ya que pueden influir diversos factores como el sobreajuste a los datos de entrenamiento. Para nuestro problema, como hemos podido observar, no es nada recomendable aplicar una arquitectura compleja puesto que proporciona resultados muy similares a la b√°sica comentada anteriormente, pero con una mayor inversi√≥n temporal y computacional.

  • Por otro lado, es importante se√±alar la mejora conseguida aplicando redes convolutivas a nuestro problema puesto que, trat√°ndose del an√°lisis y clasificaci√≥n de im√°genes, es uno de los modelos que mejores resultados ha proporcionado a lo largo de su evoluci√≥n gracias a su especializaci√≥n en este campo, como anteriormente hemos comentado.

  • Otra de las t√©cnicas aplicadas para reducir el problema del sobreaprendizaje, comentado anteriormente, consist√≠a en transformar y aumentar el n√∫mero de im√°genes para generar muestras diferentes. El objetivo consist√≠a en independizar al clasificador de las caracter√≠sticas de las im√°genes para que pudiese predecir a qu√© prenda pertenece. Como hemos podido observar, en los modelos convolutivos ha sido ventajoso y posible aplicarlo puesto que no son topolog√≠as muy complejas y los resultados han sido ligeramente mejores. De hecho, el m√©todo combinado entre un entrenamiento tradicional y otro con el conjunto aumentado durante m√°s iteraciones ha sido el que mejores resultados ha proporcionado, aunque a expensas de un cierto sacrificio en t√©rminos de tiempo de entrenamiento.

  • Por √∫ltimo destacamos que el poder utilizar un modelo pre-entrenado es una forma muy sencilla de abordar un problema de clasificaci√≥n, siempre que se le puedan a√±adir algunas capas y adaptarlas a nuestros datos. En caso contrario, como hemos podido observar, el modelo por muy bueno que sea, si el problema para el que ha sido entrenado difiere del que tenemos nosotros, no va a tener suficiente informaci√≥n como para realizar predicciones de forma adecuada. Asimismo, el tiempo de entrenamiento como podemos apreciar en la tabla resumen se dispara para estos dos modelos por lo que tambi√©n es un factor a considerar a la hora de elegir un modelo pre-entrenado.

Como conclusi√≥n final, destacamos que para nuestro problema podemos aplicar topolog√≠as muy sencillas para inmediatamente conseguir un buen clasificador. Sin embargo, si deseamos mejorar la calidad y la precisi√≥n, es preciso utilizar arquitecturas m√°s complejas, especialmente las convolutivas que se encuentran enfocadas a resolver problemas con im√°genes, pero a costa de un aumento considerable en el tiempo de entrenamiento y validaci√≥n. Es por ello por lo que para cada problema es necesario realizar un estudio de su complejidad adem√°s de ejecutar una gran variedad de experimentos para que, mediante ensayo-error, podamos ajustar los par√°metros para obtener un modelo dotado de una buena capacidad de generalizaci√≥n.
